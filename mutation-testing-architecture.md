# Mutation Testing for Haskell with sydtest

Mutation testing works by introducing small, targeted changes (mutations) to
source code, then running the test suite against each mutant. If the tests
still pass after a mutation, the mutant "survived" вҖ” indicating a gap in test
coverage. A mutant is "killed" when at least one test fails.

The implementation uses a **GHC plugin** to compile all mutations into the
binary at once and **sydtest** as the test runner. Everything runs **inside the
sydtest process** вҖ” there are no separate runner or coverage processes.

The runner operates in two phases:
1. **Coverage phase** (`--mutation-coverage <dir>`): runs each test in isolation
   to record which mutation sites it reaches; writes `<Module>.coverage.json`.
2. **Mutation phase** (`--mutation <dir>`): for each mutation, runs only the
   covering tests (or the full suite if no coverage data is available).

---

## Core Concepts

### Mutation

A mutation is a single, minimal change to an expression вҖ” e.g. replacing `+`
with `-`, `True` with `False`, `>` with `>=`.

Each mutation has a `MutationId`, generated by the plugin and written to a
manifest file alongside the build.

### Mutant activation

Rather than recompiling per mutation, all mutations are compiled into the same
binary. Each mutation site is wrapped in an `ifMutation` call:

```haskell
-- original: x + y
-- becomes:
ifMutation mutId (x - y) (x + y)
```

A single `IORef (Maybe MutationId)` is shared across the program. Only one
mutation is active at a time. When no mutation is active the overhead is one
`unsafePerformIO (readIORef activeMutation)` per mutation site.

The `IORef` is also initialised from the `MUTATION_ACTIVE` environment variable
at process start, so a mutant can be activated before `main` runs.

## Step-by-Step: What Happens and Where

### Step 1 вҖ” Build

The user's test suite is compiled with:
- `-fplugin=Test.Syd.Mutation.Plugin` вҖ” instruments every mutation site with
  `ifMutation` and injects an import of `Test.Syd.Mutation.Plugin.Runtime`
- `-fplugin-opt=Test.Syd.Mutation.Plugin:manifest=/path/to/manifest` вҖ” appends
  discovered mutation ids to the manifest file (one tab-separated line per
  mutation)

Output: compiled test binary + manifest file.

### Step 2 вҖ” Coverage phase (optional but recommended)

```
./test-binary --mutation-coverage /path/to/manifest-dir
```

`runCoverageMode`:
1. Builds the `TestForest` once.
2. For each test (identified by `TestId`), filters the forest to that test and
   runs it with `coverageSlot` installed so `ifMutation` records which
   `MutationId`s are reached.
3. Inverts the `Map TestId (Set MutationId)` to `Map MutationId (Set TestId)`.
4. Groups records by module, writes `<Module>.coverage.json` alongside the
   existing manifest files.

### Step 3 вҖ” Mutation testing

```
./test-binary --mutation /path/to/manifest-dir
```

`sydTest` detects non-empty `settingMutation` and calls `runMutationMode`.

`runMutationMode`:
1. Reads all `*.json` manifests and `*.coverage.json` coverage files from each dir.
2. Builds the `TestForest` once.
3. For each `MutationId`:
   - **No coverage data**: runs the full suite.
   - **`covering_tests: []`**: mutation is uncovered; counts as `Uncovered`.
   - **`covering_tests: [t1, t2, ...]`**: runs only those tests, filtered via `TestIdTrie`.
4. Prints `Killed: N`, `Survived: M`, and `Uncovered: K` to stdout.

The Nix check derivation fails if `Survived > 0`.

### Nix helper

```nix
checks.x86_64-linux.mutation-test = mutatieCheck {
  testSuite = haskellPackages.mypackage-test;
};
```

`mutatieCheck` wires up the test derivation: it passes `--mutation
${testSuite}/manifest` to the test binary and fails the derivation if any
mutant survived.

## Package: `sydtest-mutation-runtime`

A tiny library linked into any binary built with the plugin. Kept separate so
that instrumented test suites do not need a dependency on GHC API packages.

### `Test.Syd.Mutation.Runtime`

```haskell
newtype MutationId = MutationId [String]
  deriving (Eq, Ord, Show)

-- Process-global IORef. Initialised from MUTATION_ACTIVE env var at startup.
{-# NOINLINE activeMutation #-}
activeMutation :: IORef (Maybe MutationId)

parseMutationId  :: String      -> Maybe MutationId
renderMutationId :: MutationId  -> String

setActiveMutation :: Maybe MutationId -> IO ()
setActiveMutation = writeIORef activeMutation

-- Process-global slot for per-test coverage collection.
{-# NOINLINE coverageSlot #-}
coverageSlot :: IORef (Maybe (IORef (Set MutationId)))

withCoverageSlot :: IORef (Set MutationId) -> IO a -> IO a

-- Emitted at every mutation site by the plugin.
{-# NOINLINE ifMutation #-}
ifMutation :: MutationId -> a -> a -> a
ifMutation mid mutated original =
  unsafePerformIO $ do
    active <- readIORef activeMutation
    case active of
      Just aid -> pure $ if aid == mid then mutated else original
      Nothing -> do
        mSlot <- readIORef coverageSlot
        forM_ mSlot $ \ref -> modifyIORef' ref (Set.insert mid)
        pure original
```

`unsafePerformIO` in `ifMutation` is safe because:
- `activeMutation` and `coverageSlot` are written by the runner; the suite runs
  to completion (synchronously) before either is written again.
- `NOINLINE` prevents GHC from floating the reads out of the call site.

### `Test.Syd.Mutation.TestId`

```haskell
-- A stable identifier for a single test.
-- Each step is (description text, per-description sibling index).
newtype TestId = TestId [(Text, Int)]

renderTestId         :: TestId -> Text
parseTestIdFilterArg :: Text   -> Maybe TestId
```

## Package: `sydtest-mutation-plugin`

Depends on `sydtest-mutation-runtime` for `MutationId` and `ifMutation`.

### `Test.Syd.Mutation.Plugin`

A GHC source plugin (`Plugin` from `GHC.Plugins`). Hooks:

- **`parsedResultAction`** вҖ” injects `import Test.Syd.Mutation.Plugin.Runtime ()`
  into every instrumented module so that `ifMutation` and `MutationId` are in
  scope for the typecheck action.
- **`typeCheckResultAction`** вҖ” walks the typechecked AST; at each applicable
  node, generates a `MutationId` and wraps the original expression in
  `ifMutation mutId mutated original`; appends discovered mutations to the
  manifest file.
- **`driverPlugin`** вҖ” suppresses `-Wunused-imports` for the injected import.
- **`pluginRecompile = impurePlugin`** вҖ” always reruns (required because the
  manifest is a side effect).

The plugin skips `Paths_*` modules and any modules listed via
`--exception=<ModuleName>` options.

### `Test.Syd.Mutation.Plugin.Runtime`

Re-exports `ifMutation` and `MutationId` from `sydtest-mutation-runtime`. The
plugin injects an import of this module into each instrumented module.

### `Test.Syd.Mutation.Plugin.Instrument`

Contains the AST walking and mutation-site instrumentation logic used by
`mutationTypeCheckAction`.

## Package: `sydtest-mutation`

Library providing test-id utilities and filtered-forest running. These are used
by the mutation runner and can also be used for other selective-execution needs.

### `Test.Syd.Mutation.Forest` (in `sydtest`)

Lives in `sydtest` (not `sydtest-mutation`) so it can directly reference
`SpecDefForest`/`SpecDefTree` without a dependency cycle.
Re-exported from `Test.Syd.Mutation`.

```haskell
-- Trie over TestId paths, used for efficient filtering.
data TestIdTrie = TrieLeaf | TrieNode (Map (Text, Int) TestIdTrie)

testIdTrieFromSet  :: Set TestId -> TestIdTrie
testIdTrieFromList :: [TestId]   -> TestIdTrie

flattenTestForestWithIds :: SpecDefForest '[] () result -> [(TestId, result)]
filterTestForestByTrie   :: TestIdTrie -> TestForest '[] () -> TestForest '[] ()
```

### `Test.Syd.Mutation`

```haskell
-- Build forest + full trie in one pass.
execTestDefM' :: Settings -> Spec -> IO (TestForest '[] (), TestIdTrie)

-- Enumerate all TestIds without running tests.
getTestIds :: Spec -> IO [TestId]

-- Run a pre-built forest filtered to a trie. Returns True if all passed.
```

## Sydtest integration

### CLI flags

`Settings` in `Test.Syd.OptParse` gains:

```haskell
settingMutation         :: ![Path Abs Dir]   -- --mutation <dir>
settingMutationCoverage :: ![Path Abs Dir]   -- --mutation-coverage <dir>
```

When `settingMutationCoverage` is non-empty, `sydTest` calls `runCoverageMode`.
When `settingMutation` is non-empty (and coverage is empty), it calls `runMutationMode`.

### `runCoverageMode` / `runMutationMode` (`Test.Syd.MutationMode`)

```haskell
runCoverageMode :: Settings -> [Path Abs Dir] -> Spec -> IO ()
runMutationMode :: Settings -> [Path Abs Dir] -> Spec -> IO ()
```

Both build the spec forest once. All runs use `settingThreads = Synchronous`
because `activeMutation` and `coverageSlot` are process-global IORefs.

`runMutationMode` reads both `*.json` and `*.coverage.json` files. It outputs
three lines: `Killed: N`, `Survived: M`, `Uncovered: K`.

### Manifest files

The plugin writes `<Module>.json` to the manifest directory. Each file is a
JSON array of `MutationRecord` objects.

The coverage phase writes `<Module>.coverage.json` alongside, with the same
structure but the `covering_tests` field populated (a JSON array of
`renderTestId`-encoded `TestId`s).

`readManifestDir` reads only `*.json` (excluding `*.coverage.json`).
`readCoverageDir` reads only `*.coverage.json`.

## MutationId Format

`MutationId = [String]`. The plugin constructs this as:

```
[moduleName, operatorName, show line, show colStart, show colEnd]
```

e.g. `["MyLib.Foo", "RelationalOperator", "42", "10", "12"]`.

`sydtest-mutation-runtime` treats this as an opaque `[String]` and does not
interpret it. The format is owned entirely by `sydtest-mutation-plugin`.

In the manifest file each mutation occupies one line; fields are
tab-separated. `runMutationMode` maps tabs to `/` before calling
`parseMutationId`.

## Feedback Loops

1. **Compile runtime:** `stack build sydtest-mutation-runtime --pedantic`
2. **Compile plugin:** `stack build sydtest-mutation-plugin --pedantic`
3. **Compile mutation library:** `stack build sydtest-mutation --pedantic`
4. **Compile sydtest:** `stack build sydtest --pedantic`
5. **Run example:** `stack test sydtest-mutation-example --pedantic`
6. **CI:** `nix flake check`

## Open Questions / Future Work

- **Parallelism:** because `activeMutation` is a process-global IORef, tests
  must run serially within a single process. Parallelism could be achieved by
  spawning multiple processes each covering a disjoint partition of the
  mutation set.

- **Equivalent mutants:** some mutations produce semantically identical programs
  (e.g. `x + 0` вҶ’ `x - 0`). These are hard to detect automatically and are
  documented as a known limitation.
